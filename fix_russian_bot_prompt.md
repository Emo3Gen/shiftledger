## Задача 1/3: Починить NL-парсер русского языка

Проблема: бот не распознал "могу пн 10-13" как SHIFT_AVAILABILITY. Ответил "не распознано как команда".

1. Прочитай backend/factsParserV0.js — найди NL-парсинг русского языка.

2. Проверь и исправь: парсер ОБЯЗАН распознавать следующие фразы (все варианты):

   Доступность (SHIFT_AVAILABILITY):
   - "могу пн 10-13"
   - "могу понедельник 10-13"
   - "пн утро могу"
   - "могу в среду с 14 до 17"
   - "свободна пн 10-13"
   - "ок пн 10-13"
   - "да, могу пн 10-13"
   - "пн 10-13 могу"
   - "могу пн утро"
   - "могу пн вечер"

   Недоступность (SHIFT_UNAVAILABILITY):
   - "не могу пн 10-13"
   - "не смогу в четверг вечером"
   - "занята пн 10-13"
   - "нет пн 10-13"
   - "пн 10-13 не могу"

   Маппинг дней (регистронезависимо):
   - пн, понедельник → mon
   - вт, вторник → tue
   - ср, среда → wed
   - чт, четверг → thu
   - пт, пятница → fri
   - сб, суббота → sat
   - вс, воскресенье → sun

   Маппинг времени:
   - "утро" → from: "10:00", to: "13:00"
   - "вечер" → from: "18:00", to: "21:00"
   - "день" → from: "13:00", to: "18:00"
   - "10-13" или "10:00-13:00" или "с 10 до 13" → from: "10:00", to: "13:00"

3. Ключевые паттерны для regex (порядок важен — проверяй сначала более специфичные):

   Паттерн 1: "(могу|свободна?|ок|да)\s+<день>\s+<время>"
   Паттерн 2: "<день>\s+<время>\s+(могу|свободна?|ок)"
   Паттерн 3: "(не могу|не смогу|занята?|нет)\s+<день>\s+<время>"
   Паттерн 4: "<день>\s+<время>\s+(не могу|не смогу|занята?)"
   Паттерн 5: "(могу|свободна?)\s+в?\s*<день>\s+с?\s*<час>\s*до\s*<час>"
   Паттерн 6: "(не могу|не смогу)\s+в?\s*<день>\s+с?\s*<час>\s*до\s*<час>"

   Где <время> = утро|вечер|день|<час>-<час>|<час>:<мин>-<час>:<мин>

4. Напиши/обнови тесты в backend/__tests__/factsParserV0.test.js — добавь ВСЕ фразы из списка выше как отдельные test cases. Каждая фраза должна возвращать корректный факт.

5. Запусти npm test — все тесты должны проходить.

git commit -m "fix: NL parser now recognizes Russian availability phrases (могу пн 10-13)"
