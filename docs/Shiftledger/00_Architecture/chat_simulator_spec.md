## Chat Simulator — спецификация

UI‑инструмент для отладки полного пайплайна: **dialog → ingest → events → facts**.

### Layout (3 колонки)

- **Левая колонка: список диалогов**
  - Список диалогов по выбранному tenant’у.  
  - Элементы: `title` (chat_id или человекочитаемое имя), последний текст, время последнего сообщения, бейдж статуса.  
  - Вверху селектор tenant’а.

- **Центр: окно чата**
  - История сообщений диалога (user / bot), отсортированная по времени.  
  - Внизу input‑поле: текст, кнопка “Отправить”.  
  - При отправке:
    - вызывается `POST /debug/send`;  
    - в ленту добавляется сообщение пользователя;  
    - после ответа — сообщение бота (заглушка или реальный ответ).

- **Правая колонка: debug panel**
  - Секции:
    - `intent` — какие высокоуровневые намерения распознались (пока заглушка).  
    - `rules_version` — версия парсеров/прав, использованная для facts (например, `facts_parser: v0`).  
    - `slots` — извлечённые сущности (даты, смены, суммы и т.п.).  
    - `tool_calls` — какие внутренние вызовы/эндпоинты были задействованы (например, `/ingest`, `/facts`).  
    - `guardrails` — флаги/проверки (например, подозрительные сообщения, превышение лимитов).  
    - `latency` — тайминги (ingest, parse, round‑trip).  
    - `escalation_reason` — почему запрос может требовать внимания человека.

### API, которые вызывает Simulator

- `GET /debug/tenants`
  - Возвращает список доступных tenant’ов.  
  - Пример ответа:
    ```json
    [
      { "id": "dev", "name": "Dev Tenant" }
    ]
    ```

- `GET /debug/dialogs?tenant_id=...`
  - Возвращает список диалогов для tenant’а.  
  - Пример ответа:
    ```json
    [
      {
        "id": "dlg_1",
        "tenant_id": "dev",
        "chat_id": "debug_chat",
        "title": "debug_chat / isa",
        "last_message_at": "2026-02-06T12:00:00.000Z"
      }
    ]
    ```

- `GET /debug/dialog/:id`
  - Возвращает полное описание диалога:
    ```json
    {
      "dialog": {
        "id": "dlg_1",
        "tenant_id": "dev",
        "chat_id": "debug_chat",
        "title": "debug_chat / isa"
      },
      "events": [
        {
          "id": 123,
          "trace_id": "t-1",
          "chat_id": "debug_chat",
          "user_id": "isa",
          "text": "Пн утро могу, но с 10 до 13",
          "received_at": "2026-02-06T10:00:00.000Z"
        }
      ],
      "facts": [
        {
          "id": 1,
          "event_id": 123,
          "fact_type": "SHIFT_AVAILABILITY",
          "fact_payload": { "date": "2026-02-10", "shift": "morning", "availability": "can" },
          "parser_version": "v0",
          "status": "parsed"
        }
      ]
    }
    ```

- `POST /debug/send`
  - Эмуляция отправки сообщения от пользователя.
  - Вход:
    ```json
    {
      "tenant_id": "dev",
      "channel": "emu",
      "chat_id": "debug_chat",
      "user_id": "isa",
      "message_id": "local-1",
      "ts": "2026-02-06T10:00:00.000Z",
      "text": "Пн утро могу, но с 10 до 13",
      "meta": {
        "role": "staff"
      }
    }
    ```
  - Поведение v1:
    - Пишет событие в in‑memory storage, соответствующий диалог появляется/обновляется.  
    - (Опционально) в будущем может вызывать реальный `/ingest` и дожидаться обработки.  
  - Пример ответа (заглушка):
    ```json
    {
      "ok": true,
      "trace_id": "sim-123",
      "dialog_id": "dlg_1",
      "bot_response": {
        "text": "[stub] бот ещё не подключён",
        "meta": {
          "rules_version": "v0",
          "notes": "chat simulator stub"
        }
      }
    }
    ```

### Данные и отображаемые поля

- В списке диалогов: `title`, `chat_id`, `last_message_at`, количество фактов/ошибок (можно добавить позже).  
- В чате: помечать сообщения, для которых уже есть facts (например, иконкой).  
- В debug‑панели: показывать:
  - активный `trace_id`;  
  - список последних facts по этому dialog/event;  
  - версию парсера;  
  - собранные слоты (например, дата, смена, тип действия).
