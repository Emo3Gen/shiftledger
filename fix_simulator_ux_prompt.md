## Проблема 1: Кнопка Send не работает

1. Прочитай apps/simulator/src/App.tsx — найди обработчик кнопки "Send".
2. Открой Network tab логику: кнопка должна вызывать POST /debug/send. Проверь:
   - Правильный ли URL (через Vite proxy или абсолютный)
   - Правильный ли payload (tenant_id, chat_id, user_id, text, channel, meta)
   - Есть ли обработка ответа и ошибок
   - Обновляется ли список сообщений после отправки
3. Проверь что chat_id и user_id берутся из текущего выбранного диалога или дефолтные.
4. Проверь нет ли ошибки с auth — если API_KEY не задан, middleware пропускает, но может быть баг.
5. Добавь console.log в обработчик Send для отладки если нужно.
6. Почини проблему. Убедись что после нажатия Send:
   - Сообщение появляется в чате
   - Facts обновляются в debug panel
   - Input очищается

## Проблема 2: Юзабилити

Прочитай весь App.tsx и улучши следующее:

### Левая колонка (диалоги):
- Активный диалог должен быть визуально выделен (background color)
- Добавь кнопку "Новый диалог" (+ New Chat) — создаёт диалог с уникальным chat_id
- Последнее сообщение в превью обрезать до 50 символов

### Центральная колонка (чат):
- Input: отправка по Enter (не только по кнопке Send)
- Сообщения пользователя — справа, голубой фон. Сообщения бота — слева, серый фон
- Автоскролл вниз при новых сообщениях
- Сообщения с распознанными facts — зелёная метка или иконка ✅
- Показывать время отправки у каждого сообщения
- Кнопка Send должна быть неактивна если поле пустое

### Правая колонка (debug):
- Facts: если facts_count > 0, показывать список фактов с типом и payload (не просто число)
- Last event: показывать текст последнего события
- Schedule: после нажатия "Пересчитать график" — показывать таблицу (сетка дни/слоты как в Telegram)
- Вкладки: текущие "График", "Табель", "Незаполненные" — убедись что переключаются

### Общее:
- Все надписи на русском (если есть английские — перевести)
- Адаптивная ширина колонок (flex или grid)
- Минимальные отступы и границы для читаемости
