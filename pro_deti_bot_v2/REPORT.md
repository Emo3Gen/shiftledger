## Созданные артефакты

### Этап 1 — Реконструкция диалогов
- `sessions_full.json` — 14 023 сессии с полными сообщениями
- `sessions_meta.json` — метаданные сессий (без текстов)
- `client_profiles.json` — 1 531 профиль клиента
- `top_sessions_for_review.json` — 200 лучших сессий для ручной валидации
- `stats_report.json` — сводная статистика

### Этап 2 — База знаний
- `knowledge_base.json` — правила центра, договор оферты, информация о студии
- `intent_examples.json` — по 10 примеров на каждый интент
- `style_guide.md` — гайд по стилю общения операторов
- `rules_plain.txt` — правила в текстовом формате для промптов

### Этап 3 — Размеченный датасет
- `annotated_qa.json` — 20 524 размеченные QA-пары
- `golden_pairs.json` — 5 726 золотых пар (по интентам)
- `few_shot_examples.json` — топ-5 примеров для каждого из 12 интентов
- `entity_stats.json` — статистика по извлечённым сущностям
- `claude_batches/` — батчи для верификации через Claude API

## Следующие шаги

### Этап 4 — LLM-верификация (рекомендуется)
Прогнать 10 900 пар с интентом OTHER через Claude API Batch для точной классификации. Ориентировочная стоимость: $15-25.

### Этап 5 — Сборка бота
Адаптация ядра NEXA под детский центр:
- FSM-конфигурация для 12 интентов
- System prompt с few-shot примерами из golden_pairs
- RAG-индекс из annotated_qa (векторизация через embeddings)
- VK Bot API интеграция (основной канал клиентов)

### Этап 6 — Тестирование
- Three-judge quality system (Playbook Guardian, Style Judge, LLM Coach)
- A/B тест: бот vs реальные ответы операторов
- Регрессионные тесты на golden pairs
