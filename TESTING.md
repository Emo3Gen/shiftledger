# Тестирование ShiftLedger

## Быстрый старт

1. Запустить сервер:
   ```bash
   ./scripts/start-test.sh
   ```

2. В отдельном терминале запустить ngrok:
   ```bash
   ngrok http 3000
   ```

3. Отправить URL Семёну

## Что тестировать

### Расписание
- График на неделю: 7 дней, 2 смены (утро 10-13, вечер 18-21)
- Замены: иконка со стрелками, подсказка при наведении
- Уборки: иконка метлы в ячейках графика
- Проблемные смены: красный треугольник

### Зарплаты
- Часы: Иса 12ч, Дарина 15ч, Ксюша 12ч, Карина 3ч
- Ставка: 280 руб/ч (по умолчанию)
- Уборки: 500 руб за уборку
- Доп. занятия: 500 руб базовая + 100 руб за каждого ребёнка свыше 8
- Итого: ~42ч, ~17500 руб

### Настройки
- Сотрудники: редактирование (имя, роль, ставка, телефон)
- Ставки: inline-редактирование
- Смены: шаблоны слотов

### Workflow кнопок
- Начать сбор (OPEN_WEEK) — доступна когда неделя CLOSED или нет данных
- Собрать график (BUILD_SCHEDULE) — доступна в состоянии COLLECTING
- Закрыть неделю (WEEK_LOCK) — доступна в состоянии ACTIVE

### Навигация по неделям
- Стрелки влево/вправо
- Кнопка "Сегодня"
- Индикатор "есть данные" / "нет данных"

### Отладочные кнопки в панели
- **Сбросить неделю** (красная) — удаляет все факты и события за текущую неделю, с подтверждением
- **Загрузить тестовый сценарий** (голубая) — прогоняет Сценарий Б для текущей недели

## Тестовые данные

Загружаются автоматически при первом запуске для текущей недели.

Для сброса и перезагрузки:
```bash
curl -X POST http://localhost:3000/debug/seed
```

Или кнопка "Загрузить тестовые данные" на экране приветствия.

Для сброса конкретной недели:
```bash
curl -X POST http://localhost:3000/debug/reset-week -H 'Content-Type: application/json' -d '{"week_start":"2026-02-23"}'
```

## Endpoints

- Панель директора: `http://localhost:3000/#/director`
- API документация: `http://localhost:3000/api-docs`
- Health check: `http://localhost:3000/health`

## Тестирование в Telegram

### Подготовка

1. Бот запускается автоматически вместе с сервером (long polling)
2. Токен уже настроен в `backend/.env.dev`
3. Создать группу в Telegram, добавить бота (дать права администратора)

### Команда /as — смена роли (для тестирования)

Вместо привязки аккаунтов через `/link`, в dev-режиме можно быстро переключаться между сотрудниками:

```
/as Иса     — все сообщения идут как от Исы (u1)
/as Дарина  — переключает на Дарину (u2)
/as Ксюша   — переключает на Ксюшу (u3)
/as Карина  — переключает на Карину (u4)
/as сброс   — вернуться к своему аккаунту
/as         — показать текущую роль
```

Один человек может тестировать за нескольких сотрудников!

### Команды бота

| Команда | Алиас | Описание |
|---------|-------|----------|
| `/start` | — | Приветствие и справка |
| `/help` | `помощь` | Справка по командам |
| `/schedule` | `расписание` | График на неделю |
| `/status` | `статус` | Состояние недели (сбор/активна/закрыта) |
| `/pay` | `зарплата` | Зарплата текущего сотрудника |
| `/link` | — | Привязка Telegram к сотруднику (постоянная) |
| `/as` | — | Временная смена роли (только dev) |

### Сценарий полного теста вдвоём

**Подготовка:**
1. Создать группу в Telegram, добавить бота (админ-права для закрепления)
2. В панели директора: **Сбросить неделю** (если есть старые данные)
3. В панели: **Начать сбор**

**Сбор доступности (в группе):**
```
/as Иса
могу пн утро, вт утро, чт утро, пт утро, сб утро

/as Дарина
могу пн вечер, вт вечер, ср утро, пт вечер, вс вечер

/as Ксюша
могу ср вечер, чт утро, чт вечер, пт вечер, вс утро

/as Карина
могу пн вечер, сб вечер
```

**Сборка графика:**
4. В панели: **Собрать график**
5. Проверить расписание — все смены должны быть распределены
6. В Telegram: `/schedule` — сравнить с панелью

**Замена:**
```
/as Иса
не могу чт утро, кто подменит?

/as Ксюша
я подменю Ису в чт утро
```

7. В панели: проверить замену (иконка со стрелками)

**Уборки и доп.занятия (в Telegram):**
```
/as Дарина
уборка вт

/as Иса
доп.занятие пн 12 детей
```

**Закрытие:**
8. В панели: **Закрыть неделю**
9. Проверить зарплаты — вкладка "Зарплаты"
10. В Telegram: `/pay` (от имени каждого сотрудника через `/as`)

### Что проверить

- [ ] `/as Иса` → бот отвечает "Теперь вы пишете за Ису"
- [ ] Сообщения доступности парсятся и создают факты
- [ ] `/schedule` показывает расписание
- [ ] `/status` показывает состояние недели
- [ ] `/pay` показывает зарплату привязанного сотрудника
- [ ] Закреплённое сообщение обновляется после каждого факта
- [ ] Замена отображается в панели с иконкой
- [ ] Кнопка "Сбросить неделю" удаляет все данные
- [ ] Кнопка "Загрузить тестовый сценарий" создаёт полный набор данных
