Проанализируй и восстанови Chat Simulator (apps/simulator/).

## Что сделать

1. Прочитай apps/simulator/ — изучи все файлы, компоненты, конфигурацию. Определи:
   - На чём написан (React/Vite/etc.)
   - Какие компоненты есть
   - Какие API-вызовы делает к бэкенду
   - Что работает, что сломано

2. Прочитай docs/chat_simulator_spec.md (или корневой chat_simulator_spec.md) — сверь текущий код со спецификацией.

3. Убедись что бэкенд отдаёт CORS-заголовки для localhost фронтенда. Если нет — добавь cors middleware в server.js.

4. Установи зависимости фронтенда:
   cd apps/simulator && npm install

5. Попробуй запустить: npm run dev (или npm start). Если не запускается — почини ошибки.

6. Проверь что все 3 колонки работают:
   - Левая: список tenant'ов (GET /debug/tenants) и диалогов (GET /debug/dialogs)
   - Центр: окно чата с историей и input-полем (GET /debug/dialog/:id, POST /debug/send)
   - Правая: debug panel (facts, parser version, slots, schedule)

7. Если какие-то API-вызовы не совпадают с текущими эндпоинтами бэкенда — обнови фронтенд или бэкенд для совместимости.

8. Убедись что отправка сообщения через симулятор:
   - Вызывает POST /debug/send
   - Показывает сообщение в чате
   - Показывает распознанные facts в debug panel
   - Показывает расписание

9. Если симулятор работал раньше но сломался из-за изменений в Волнах 1-3 (auth middleware, новые эндпоинты, переименования) — адаптируй его.

10. Добавь в README инструкцию по запуску симулятора.

git commit -m "fix: restore and update Chat Simulator UI"

## Финал
- Выведи URL симулятора (например http://localhost:5173)
- Выведи список починенных/обновлённых файлов
- Краткий отчёт: что работало, что починил, что осталось
